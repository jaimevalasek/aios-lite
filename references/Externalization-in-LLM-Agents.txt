Externalization in LLM Agents: A Systems-Level Briefing on Cognitive Infrastructure

Executive Summary

The development of Large Language Model (LLM) agents has undergone a fundamental architectural shift. Progress is no longer driven solely by increasing model parameters ("Weights") or refining prompts ("Context"), but by relocating cognitive burdens into external infrastructure ("Harness Engineering"). This process, termed externalization, transforms difficult internal cognitive tasks into forms that models can solve more reliably.

The core of modern agent design rests on three externalization dimensions coordinated by a central harness:

* Memory (Externalized State): Converts the burden of temporal continuity from internal recall to selective retrieval and recognition.
* Skills (Externalized Expertise): Transforms procedural know-how from improvised generation into the selection and composition of pre-validated routines.
* Protocols (Externalized Interaction): Replaces ad-hoc communicative inference with structured, machine-readable contracts for tools and multi-agent coordination.

Reliable agency now emerges from the coupling of the model with a designed "cognitive environment." This shift mirrors human cognitive history—moving from thought to language, writing, and digital computation—where the use of artifacts does not change internal capability but fundamentally reorganizes the task itself.
--------------------------------------------------------------------------------
1. The Historical Progression: From Weights to Harness

The trajectory of LLM agent capability is characterized by three distinct layers of investment and engineering effort:

Capability Layer	Period	Focus	Key Mechanisms
Weights	2022	Parametric knowledge	Pretraining, SFT, RLHF, Scaling Laws
Context	2023–2024	Input engineering	Prompting, RAG, Chain-of-Thought, Iterative Self-Critique
Harness	2025–2026	Infrastructure	Memory stores, Skill registries, Protocols, Sandboxing

The Limits of Weights and Context

* Weights: Parametric knowledge is static, difficult to update selectively, and lacks the ability to differentiate between millions of users at the parameter level.
* Context: While flexible, context windows are finite and costly. Models suffer from the "lost in the middle" phenomenon, where retrieval accuracy drops for information placed in the center of long contexts.

Externalization as the Transition Logic

Externalization is the mechanism explaining this architectural shift. Drawing on Donald Norman’s theory of cognitive artifacts, the document argues that external aids do not merely amplify ability; they restructure the problem. For example, a shopping list transforms a recall problem into a recognition problem. Similarly, externalizing an agent's state, expertise, and interactions allows the model to spend its fixed inferential capacity on reasoning rather than remembering or improvising syntax.
--------------------------------------------------------------------------------
2. Dimension 1: Externalized State (Memory)

Memory externalization addresses the "continuity problem." It allows accumulated knowledge to persist beyond a single session, moving the burden of history out of the ephemeral context window.

The Four Dimensions of Externalized State

1. Working Context: The live intermediate state of a task (e.g., open files, active hypotheses).
2. Episodic Experience: Records of prior runs, including failures and reflections, used as precedents.
3. Semantic Knowledge: Abstractions that outlive episodes, such as domain facts and project conventions.
4. Personalized Memory: Stable information about specific users, habits, and preferences, subject to distinct privacy and retention rules.

Evolution of Memory Architectures

* Monolithic Context: All history remains in the prompt; scales poorly.
* Context with Retrieval Storage: Uses flat similarity search (RAG) to inject documents at query time.
* Hierarchical Memory & Orchestration: Borrows from operating systems to manage "hot" working state and "cold" long-tail storage (e.g., MemGPT, MemoryOS).
* Adaptive Memory Systems: Uses feedback-based strategies (RL or Mixture-of-Experts) to optimize retrieval and consolidate fragmented experience into abstract knowledge.
--------------------------------------------------------------------------------
3. Dimension 2: Externalized Expertise (Skills)

Skill externalization addresses the "variance problem." Instead of regenerating workflows on every invocation, expertise is packaged into reusable artifacts.

Components of a Skill Artifact

* Operational Procedures: The task skeleton—decomposition into steps, dependencies, and stopping conditions.
* Decision Heuristics: Practical rules of thumb for branching decisions and local policy (e.g., when to back off or escalate).
* Normative Constraints: Safety boundaries, compliance rules, and operational limits.

The Skill Lifecycle

The document identifies a shift from "Execution Primitives" (atomic tool calls) to "Capability Packages" (structured procedural know-how).

* Acquisition: Skills are authored by experts, distilled from successful memory traces, or discovered through environmental exploration (e.g., Voyager).
* Activation Pipeline: Involves discovery via registries, progressive disclosure (loading details only when needed), and binding the skill to executable substrates like APIs or sub-agents.
--------------------------------------------------------------------------------
4. Dimension 3: Externalized Interaction (Protocols)

Protocols address the "coordination problem." They replace ambiguous communicative inference with explicit machine-readable contracts.

Key Protocol Families

* Agent-Tool (e.g., MCP): Standardizes how agents discover and invoke tools across heterogeneous services, decoupling ecosystems from specific model providers.
* Agent-Agent (e.g., A2A, ANP): Governs how agents delegate tasks, negotiate, and exchange state, enabling open agentic webs.
* Agent-User (e.g., A2UI, AG-UI): Formalizes how execution state and interface structures are exposed to humans for consistent rendering.
* Domain-Specific: Vertical protocols for high-risk workflows like payments (AP2) and commerce (UCP).
--------------------------------------------------------------------------------
5. The Unifying Layer: Harness Engineering

A harness is the runtime environment that coordinates externalized modules into coherent agency. It sets the agent’s "practical cognitive boundary."

The Six Dimensions of Harness Design

1. Agent Loop and Control Flow: Manages the perceive-retrieve-plan-act-observe cycle and enforces recursion/cost bounds.
2. Sandboxing and Isolation: Creates controlled execution boundaries to limit side effects and ensure reproducibility.
3. Human Oversight: Configures approval gates and "hooks" for intervention.
4. Observability: Provides structured logging and execution traces for auditing and internal feedback loops.
5. Configuration and Policy: Encodes permissions and governance rules across user, project, and organizational scopes.
6. Context Budget Management: Dynamically allocates the scarce token budget between memory, skills, and protocol schemas.
--------------------------------------------------------------------------------
6. Cross-Cutting Analysis: Module Couplings

The power of a harnessed agent arises from the interactions between memory, skills, and protocols.

Interaction Path	Process	Result
Memory → Skill	Experience Distillation	Successful trajectories become codified, reusable expertise.
Skill → Memory	Execution Recording	Outcomes and failures are stored to refine future skill selection.
Skill → Protocol	Capability Invocation	Abstract procedures are grounded in governed, executable actions.
Protocol → Skill	Capability Generation	Standardized interfaces provide the structure to author new skills.
Memory → Protocol	Strategy Selection	Historical success rates inform which interaction path to choose.
Protocol → Memory	Result Assimilation	External state transitions are normalized and written back to state.
--------------------------------------------------------------------------------

7. Emerging Directions and Risks

Embodied Externalization

The same logic is extending to robotics. The "Cerebrum-Cerebellum split" separates high-level LLM-based planning (cerebrum) from specialized, low-latency visuomotor policies (cerebellum). This allows the high-level agent to orchestrate physical skills through protocolized status reports.

Security and Governance

As cognition moves outward, the attack surface expands:

* Memory Poisoning: Distorting future reasoning through corrupt traces.
* Skill Injection: Embedding adversarial procedures into reusable repertoires.
* Lethal Trifecta: The dangerous combination of sensitive data access, unconstrained communication, and unverified execution.

Future Outlook

The frontier is moving toward Self-Evolving Harnesses, where the system adapts its own orchestration logic via reinforcement learning or program synthesis. Ultimately, externalization is shifting from private scaffolding to shared infrastructure, where agents across an ecosystem can contribute to and draw from collective memory and skill repositories.
