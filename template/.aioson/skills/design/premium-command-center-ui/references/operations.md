# Operations

## Hierarchy rules

1. Show live pressure first.
2. Group cards by operational meaning, not by raw data type.
3. Convert backend shapes into UI-native view models when necessary.
4. Expose the next useful action from every major block.

## When it is working

The result should feel like an operating surface:
- search and navigation reduce friction
- contextual rails support the current task
- signals, queues, history, and drill-downs coexist cleanly
