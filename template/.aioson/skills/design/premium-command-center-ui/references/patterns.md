# Patterns

## Primary shell

Use a tri-rail shell only when the product has enough operational depth to justify it:
- left navigation rail
- central workspace
- right contextual activity rail

## Page archetypes

- command overview
- queue / control tower
- grouped registry
- workflow catalog
- knowledge explorer

## Guardrails

- Do not create equal-weight card walls.
- The screen needs one obvious focal block.
- Context belongs in side rails, not in the main narrative column.
