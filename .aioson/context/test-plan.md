## Strategy Chosen
**Risk-first Gap Filling** with conformance contract as the structured specification.

## Justification
- The project has 149 existing tests covering ~110 of 246 source files.
- The most recent feature (`pentester-agent`, Phase 1–4 complete) modified 4 core modules but only `workflow-next.js` has existing test coverage.
- The conformance contract `conformance-pentester-agent.yaml` defines 8 acceptance criteria that map directly to testable behaviors.
- Per `discovery.md` business rules, auth/governance, spec/state management, and core context utilities are the highest-risk uncovered areas.
- We do NOT use bottom-up pyramid here because the codebase already has a substantial test base; the priority is to close high-risk gaps, not restart from zero.

## Bugs Found During Testing

### [bug-found-001] `src/constants.js` — Missing `pentester.md` in `MANAGED_FILES` ✓ FIXED
- **Module:** `src/constants.js`
- **Test:** `MANAGED_FILES includes agent files for all defined agents`
- **Expected:** `.aioson/agents/pentester.md` should be present in `MANAGED_FILES` because `AGENT_DEFINITIONS` includes the `pentester` agent.
- **Actual:** `MANAGED_FILES` did not include `.aioson/agents/pentester.md` or `.aioson/agents/tester.md`.
- **Fix:** Added `.aioson/agents/pentester.md` and `.aioson/agents/tester.md` to `MANAGED_FILES` array.
- **Validated:** `constants-pentester.test.js` 8/8 pass.

### [bug-found-002] `src/commands/workflow-next.js` — `parseFeaturesMarkdown` includes markdown table separator as a feature ✓ FIXED
- **Module:** `src/commands/workflow-next.js`
- **Test:** `parseFeaturesMarkdown extracts feature rows correctly`
- **Expected:** Only actual feature rows should be parsed; table separator lines (`|------|--------|...|`) should be skipped.
- **Actual:** `parseFeaturesMarkdown` returned the separator line as a feature with `slug: "------"`.
- **Fix:** Added `.filter((row) => !/^-+$/ .test(row.slug))` to skip separator rows.
- **Validated:** `workflow-next-pentester.test.js` 12/12 pass.

### [bug-found-003] `src/commands/qa-init.js` — `extractFrontmatterValue` only parses list-style frontmatter (`- key: value`), not pure YAML ✓ FIXED
- **Module:** `src/commands/qa-init.js`
- **Test:** `reads URL from project.context.md list-style frontmatter`
- **Expected:** `extractFrontmatterValue` should extract `app_url` from standard YAML frontmatter blocks.
- **Actual:** The regex only matched list-item lines, so standard YAML frontmatter was ignored.
- **Fix:** Updated `extractFrontmatterValue` to try list-style first, then fallback to pure YAML (`extractYamlValue` logic).
- **Validated:** `qa-init.test.js` 6/6 pass.
