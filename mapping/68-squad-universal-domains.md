## A Correção de Rota

O Squad Genome + Mentes não é um sistema para desenvolvimento de software.
É um sistema para **qualquer trabalho que exige perspectiva especializada**.

O equívoco dos exemplos anteriores: usamos "laravel", "fintech", "dev" como exemplos.
Isso fez o sistema parecer um add-on técnico.

**A verdade:** o sistema é radicalmente agnóstico de domínio.

```
Genoma  =  pacote de conhecimento especializado em QUALQUER área
Mente   =  perspectiva de pensamento universal, funciona em QUALQUER contexto
Squad   =  combinação que cria um time especialista para QUALQUER objetivo
```

O que muda entre domínios é o **Genoma** (conhecimento específico).
O que permanece são as **Mentes** (perspectivas de pensamento).

O Cético que questiona código é o mesmo Cético que questiona um roteiro de YouTube.
O Estrategista que pensa em crescimento de software pensa em crescimento de canal.

---

## O @squad como Meta-Agente Universal

O agente @squad não é especialista em nada.
Ele é especialista em **criar especialistas**.

```
Usuário: "Quero um squad para criar conteúdo para YouTube"

@squad:
1. Entende o domínio: produção de conteúdo + plataforma YouTube
2. Identifica o que o squad precisa SABER (genomas)
3. Identifica como o squad precisa PENSAR (mentes)
4. Gera os arquivos do squad
5. O squad fica ativo e pronto para trabalhar

Nenhuma limitação de domínio. Nenhuma lista pré-definida de áreas suportadas.
```

Se o usuário pede um squad para criar cardápios de restaurante, @squad entende,
pesquisa o domínio (gastronomia, psicologia de menu, pricing) e gera o squad.
O sistema não precisa ter um "genoma:gastronomia" pré-existente — @squad cria um.

## Exemplos de Squads Fora de Tech

### YouTube Content Squad

**Domínio:** Criação de conteúdo para YouTube

**Genomas gerados:**
```markdown
# Genome: youtube-algoritmo
- CTR médio por nicho (benchmarks reais)
- Retenção: o que faz manter vs abandonar
- Títulos: fórmulas que funcionam (curiosidade, promessa, contrarian)
- Thumbnails: rostos > paisagens, contraste, texto < 6 palavras
- SEO do YouTube: tags, descrição, capítulos, cards
- Shorts vs Long-form: quando usar cada formato
- O que o algoritmo penaliza: clickbait sem entrega, baixa retenção inicial

# Genome: roteiro-video
- Estrutura de hook (primeiros 30 segundos decidem tudo)
- Pattern interrupt: trocar de cena, estímulo visual a cada 90-120s
- Promessa → Entrega → Próximo vídeo
- Calls to action: quando e como pedir like/inscrição sem parecer implorar
- Pacing: ritmo de edição por nicho (tech vs lifestyle vs educação)
- B-roll: o que mostra, o que não mostra
- Storytelling: abertura em mídia res vs contexto primeiro

# Genome: thumbnail-design
- Psicologia de contraste: o thumbnail compete com outros 8 na tela
- Faces com emoção extrema superam paisagens em quase todos os nichos
- Curiosity gap: o que está faltando que o título não entrega
- Regra dos terços aplicada a thumbnails
- Paletas que funcionam por nicho (tech: azul/preto; lifestyle: warm tones)
- Texto: máximo 5 palavras, fonte bold, contorno para legibilidade mobile
- O que NUNCA fazer: rosto + texto sobre o rosto
```

**Mentes ativadas:**
```markdown
O Espectador: "Eu clicaria nisso? Eu assistiria até o fim?"
O Algoritmo: "O YouTube vai distribuir isso para quem? Em qual momento?"
O Crítico: "O que está errado neste conceito antes de produzir?"
O Estrategista: "Como este vídeo serve ao canal em 6 meses?"
```

**Squad em ação:**
```
Usuário: "Quero criar um vídeo sobre como aprender inglês em 3 meses"

O Espectador: "Este título é genérico. Já vi 500 vídeos assim.
               O que faz o SEU diferente? Qual transformação específica você promete?"

O Algoritmo: "Alto CPC (inglês = anunciantes pagam bem). Mas competição brutal.
              Ângulo contrarian funciona melhor: 'Aprendi inglês SEM pagar curso'
              — promessa específica + desafio ao senso comum."

O Crítico: "'3 meses' varia muito por pessoa. Quem não conseguir culpa o vídeo.
            Prefira prometer o método, não o prazo."

O Estrategista: "Este é topo de funil. O próximo vídeo já deve estar planejado:
                 'Os 5 recursos gratuitos que uso todo dia' — retém quem engajou."

@roteirista (gerado pelo squad) recebe tudo isso e cria com contexto completo.

Resultado: título ajustado, thumbnail conceituada, roteiro com hook testado,
           estratégia de série definida. Antes de gravar um segundo de vídeo.
```

## Como o @squad Gera um Squad para Qualquer Domínio

O processo é sempre o mesmo, independente do domínio:

```
1. ENTENDER
   @squad pergunta: "O que este squad vai produzir / resolver?"
   Usuário: "Criar conteúdo para YouTube sobre finanças pessoais"

2. MAPEAR O DOMÍNIO
   @squad identifica:
   - Conhecimentos necessários (genomas)
   - Perspectivas que enriquecem (mentes)
   - Entregáveis esperados (o que o squad produz)

3. GERAR OU CARREGAR
   - Genomas: existem na biblioteca? Carrega. Não existem? Cria via pesquisa.
   - Mentes: seleciona os arquétipos mais relevantes para este domínio
   - Combina em squad.md

4. ATIVAR
   Os agentes do squad ficam disponíveis como perspectivas consultáveis
   em qualquer conversa daquele projeto
```

## Como Mentes São Verdadeiramente Universais

A prova de que uma Mente é bem construída: ela funciona em qualquer domínio.

```
Mente: O Cético

Em desenvolvimento:   "Este endpoint tem autorização verificada?"
Em YouTube:           "Alguém já fez este vídeo melhor que você vai fazer?"
Em jurídico:          "Qual a interpretação mais desfavorável desta cláusula?"
Em gastronomia:       "O cliente vai entender que este prato justifica R$ 85?"
Em marketing:         "Por que alguém NÃO compraria depois de ver este anúncio?"

Mesma perspectiva. Cinco domínios completamente diferentes.
```

Isso confirma que Mentes e Genomas são separados por design.
Misturar os dois (como aiox-core faz ao clonar uma pessoa inteira)
cria um agente que só funciona no contexto daquela pessoa.

Separar cria flexibilidade ilimitada.

## A Biblioteca que Cresce com a Comunidade

Com domínio-agnóstico, a biblioteca de genomas da comunidade explode:

```bash
aios-lite squad:import github:fulano/genome-copywriting-br
aios-lite squad:import github:beltrano/genome-youtube-faceless
aios-lite squad:import github:ciclano/genome-ecommerce-meli
aios-lite squad:import github:alguem/squad-startup-pitch-deck
```

Cada criador de conteúdo, advogado, chef, arquiteto que usa aios-lite
pode exportar o genoma do seu domínio e compartilhar.

O valor da rede cresce exponencialmente:
mais domínios → mais genomas → mais squads possíveis → mais usuários.
