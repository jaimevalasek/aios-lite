Conceito

O que é um Squad?

Um Squad é um time de agentes especializados que colabora para completar tarefas complexas de forma autônoma. Cada agente tem um papel, ferramentas, e um contrato de entrega claro.

Estrutura de um Squad

🎯 Objetivo do Squad
↓
🧠 Orquestrador
↙    ↓    ↘
⚙️ Worker A
⚙️ Worker B
⚙️ Worker C
↓ bus de mensagens ↓
📦 Output
📊 STATE.md
🧠 Learnings

O orquestrador decompõe o objetivo em tarefas, distribui para os workers em paralelo (por waves), monitora o progresso via bus de mensagens, e lida com falhas automaticamente — sem você precisar intervir.

✓

Diferença-chave: um Squad não é um único agente fazendo tudo. São especialistas trabalhando em paralelo, com contexto isolado por worker, comunicando-se por mensagens.

Decisão

Quando usar um Squad?

✓ Use Squad quando...

A tarefa tem partes independentes que podem rodar em paralelo

Especialistas diferentes precisam contribuir (pesquisador, escritor, revisor)

O trabalho se repete em sessões futuras

Você quer que o resultado melhore com o tempo via learnings

✗ Use agente único quando...

A tarefa é linear e curta (< 30 min)

Um único agente (/dev, /qa) resolve o problema

Não há paralelismo real possível

É exploração ou prototipagem rápida

Estrutura

Anatomia de um Squad

Todo squad criado pelo AIOSON tem esta estrutura de arquivos:

# Gerado automaticamente por aioson squad:scaffold .aioson/squads/meu-squad/ ├── agents/ │ ├── agents.md # roster de executores │ ├── orquestrador.md # agent do coordenador │ ├── worker-a.md # agent do worker A │ └── worker-b.md # agent do worker B ├── squad.manifest.json # configuração principal ├── squad.md # documentação do squad ├── STATE.md # memória cross-session ├── docs/ │ ├── design-doc.md # decisões de arquitetura │ └── readiness.md # checklist de prontidão ├── checklists/ │ └── quality.md # critérios de qualidade ├── learnings/ │ └── index.md # aprendizados acumulados └── bus/ # mensagens inter-worker (JSONL) output/meu-squad/ # outputs gerados aioson-logs/meu-squad/ # logs de sessão

O squad.manifest.json

O coração de um squad. Define executores, budget, dependências e comportamento:

{ "slug": "meu-squad", "name": "Meu Squad de Conteúdo", "mode": "content", // content | code | hybrid "executors": [ { "slug": "researcher", "title": "Pesquisador", "role": "Pesquisa tendências e fontes", "type": "agent", "modelTier": "balanced" // fast | balanced | powerful }, { "slug": "writer", "title": "Escritor", "role": "Escreve o conteúdo final", "type": "agent", "modelTier": "powerful" } ], "budget": { "max_tokens_per_session": 200000, "max_tokens_per_task": 20000, "action_on_exceed": "pause" // pause | abort | notify }, "depends_on": [], // dependências de outros squads "subscriptions": [] // eventos de outros squads para ouvir }
Prática

Criando seu primeiro Squad

O agente /squad guia você por todo o processo. Siga estes passos:

1

Ative o agente @squad

No Claude Code, digite /squad e descreva o que você quer construir. Seja específico sobre o domínio e o objetivo.

/squad # Exemplos de boas descrições: "Quero um squad para produzir roteiros de YouTube sobre tecnologia, pesquisando tendências, escrevendo o roteiro e revisando antes de publicar" "Squad de desenvolvimento para criar features em Node.js — preciso de um arquiteto, developer e QA" "Time de pesquisa jurídica para analisar contratos e identificar riscos"
2

Revise o design (blueprint)

O agente apresenta um blueprint do squad antes de criar qualquer arquivo. Você pode aceitar, ajustar papéis, ou pedir mais especialistas.

💡

Este passo é intencional — decisões sobre papéis e ferramentas impactam todas as sessões futuras. Vale 2 minutos de revisão.

3

Squad é criado automaticamente

Com o blueprint aprovado, o agente executa o scaffolding e gera todos os arquivos:

# Gerado internamente pelo @squad: aioson squad:scaffold . --slug=youtube-roteiros --name="YouTube Roteiros" --mode=content # Resultado: ✓ .aioson/squads/youtube-roteiros/ (15 arquivos criados) ✓ output/youtube-roteiros/ ✓ aioson-logs/youtube-roteiros/
4

Agentes especializados são gerados

Cada executor recebe seu próprio arquivo .md com papel, ferramentas, contexto e critérios de qualidade específicos para o domínio.

5

Squad registrado no projeto

Os atalhos são adicionados ao CLAUDE.md e AGENTS.md, e você já pode invocar os agentes individuais.

# Adicionado ao CLAUDE.md automaticamente: ## Squad: YouTube Roteiros - /pesquisador -> .aioson/squads/youtube-roteiros/agents/pesquisador.md - /escritor -> .aioson/squads/youtube-roteiros/agents/escritor.md - /revisor -> .aioson/squads/youtube-roteiros/agents/revisor.md - /orquestrador -> .aioson/squads/youtube-roteiros/agents/orquestrador.md
Componentes

Tipos de Executor

Cada worker em um squad tem um tipo que define como ele executa suas tarefas:

Tipo Quando usar Model tier padrão
agent Worker principal — recebe briefing, usa ferramentas, entrega output balanced
worker Executa scripts determinísticos (Node.js, Python) — sem LLM none
reviewer Avalia output de outros workers com critérios estruturados balanced
research Pesquisa web com cache automático (7 dias) powerful
human-gate Ponto de aprovação humana — pausa execução até confirmação human
clone Múltiplas instâncias do mesmo agente rodando em paralelo (voting) balanced
Execução

Execução Autônoma com squad:autorun

Após criar o squad, você pode rodá-lo sem supervisão. O squad:autorun cuida de tudo:

# Execução básica aioson squad:autorun . --squad=youtube-roteiros --goal="Criar roteiro sobre IA generativa em 2026" # Com validação de dependências de outros squads aioson squad:autorun . --squad=distribuicao --goal="Publicar episódio" --wait-deps # Com engine nativo do Claude Code (experimental) aioson squad:autorun . --squad=youtube-roteiros --goal="..." --engine=agent-teams # Em modo daemon (roda em background continuamente) aioson squad:daemon . --squad=youtube-roteiros --persistent

Proteções automáticas

🔄 Gap Closure Loop

Se uma tarefa falha, o sistema retenta até 3x com o contexto do erro anterior — não é retry burro, é retry informado.

💓 Heartbeat

Workers longos (>30s) enviam pulso a cada 30 segundos. Sem resposta em 3 pulsos = task considerada travada.

🛑 Anti-Loop Guard

Se um worker envia 8+ mensagens de status sem entregar resultado, o coordenador intervém e força uma decisão.

💰 Budget Gate

Antes de cada task, o sistema projeta o uso de tokens. Se a projeção excede o budget configurado, pausa e avisa.

Comunicação

Bus de Mensagens Intra-Squad

Workers se comunicam via um bus de mensagens JSONL — o coordenador observa todo o tráfego e age quando necessário.

Tipo Significado Quem responde
result Worker concluiu uma tarefa com sucesso Coordenador avança o plano
block Worker está travado e precisa de ajuda Coordenador resolve ou escala
question Worker tem dúvida sobre escopo ou decisão Coordenador ou peer worker
finding Worker descobriu algo relevante para os outros Outros workers leem se pertinente
heartbeat Worker ainda está ativo (tasks longas) Coordenador monitora ausência
resolution Um block foi resolvido Worker bloqueado retoma
📋

O bus é persistido em .aioson/squads/{slug}/bus/*.jsonl. Você pode inspecioná-lo para depurar sessions e entender o que aconteceu.

Orquestração

Conectando Squads

Squads diferentes no mesmo projeto podem se comunicar via eventos pub/sub. O squad A publica eventos quando conclui trabalho; o squad B os consome automaticamente no início da próxima run.

# Declare dependências no squad.manifest.json do squad B: { "slug": "squad-distribuicao", "depends_on": [ { "squad": "squad-conteudo", "event": "episodio.criado" // aguardar este evento } ], "subscriptions": ["episodio.*"] // padrão de eventos para ouvir } # Executar squad B — aguarda evento automaticamente (até 60s) aioson squad:autorun . --squad=squad-distribuicao --goal="Publicar" --wait-deps # Visualizar o grafo de dependências entre todos os squads aioson squad:dependency-graph .
⚠️

Use --ignore-deps para executar sem validar dependências — útil em desenvolvimento, mas não em produção.

Evolução

Memória e Aprendizado

O squad melhora com o tempo. A cada sessão, learnings são extraídas automaticamente e persistidas para influenciar sessões futuras.

1

STATE.md — memória cross-session

Mantém decisões tomadas, blockers encontrados, pendências e velocidade do squad. Lido no início de cada run para dar contexto ao orquestrador.

# .aioson/squads/{slug}/STATE.md squad: youtube-roteiros sessions_completed: 12 avg_tasks_per_session: 4.2 ## Decisions Made - Usar formato JSON para output (integração webhook) - Roteiros entre 8-12 min performam melhor ## Active Blockers - API do YouTube precisa de nova key (expirou)
2

Agent Memory — memória por executor

Cada executor acumula conhecimento específico do seu domínio. Um pesquisador aprende quais fontes são mais confiáveis; um escritor aprende quais formatos performam melhor.

# .aioson/squads/{slug}/agent-memory/escritor.md ## Patterns that work - Hooks de curiosidade nos primeiros 15s aumentam retenção 35% - Roteiros < 10 min têm 2x mais completion rate ## Patterns to avoid - Intros genéricas > 30s causam 60% de drop-off
3

Learning Evolution — aprendizado aplicado

Learnings com frequência ≥ 2 podem ser promovidas para regras do squad, modificando automaticamente o comportamento futuro.

# Detectar candidatos a automação aioson pattern:detect . --squad=youtube-roteiros # Aplicar learnings como regras aioson learning:evolve . --squad=youtube-roteiros --auto-apply # Reverter uma evolução indesejada aioson learning:rollback . --squad=youtube-roteiros --evolution=uuid-aqui

Referência de Comandos CLI

Criação e setup

ComandoO que faz
squad:scaffoldGera todos os diretórios e arquivos skeleton de um squad (sem LLM)
brief:validateValida completude de um brief antes de spawnar workers — detecta campos faltando e pode auto-corrigir
preflight:contextEstima budget de tokens antes de iniciar uma sessão — alerta sobre risco de truncação

Execução

ComandoO que faz
squad:autorunExecução autônoma completa — decompõe goal, executa em waves, verifica resultados
squad:daemonModo persistente — fica em loop monitorando eventos e rodando sessões automaticamente
squad:statusMostra estado atual do squad (STATE.md + eventos pendentes)

Análise e evolução

ComandoO que faz
pattern:detectDetecta candidatos a automação analisando learnings e histórico de sessões
learning:evolvePromove learnings frequentes para regras do squad
learning:rollbackReverte uma evolução aplicada por UUID
context:compactCompacta contexto de sessão longa em handoff estruturado
squad:dependency-graphVisualiza dependências entre squads (ASCII ou Mermaid)

Integração e exportação

ComandoO que faz
squad:cardGera Agent Card compatível com Google A2A v1.0 — permite que outros sistemas descubram o squad
agent:export-skillExporta um agente como Agent Skills Standard — portável para Cursor, Copilot, Gemini CLI e mais
